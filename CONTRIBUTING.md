# Contributing to CTTA Survey

Thank you for your interest in contributing to the **Continual Test-Time Adaptation (CTTA)** survey! We welcome new papers, code implementations, and corrections.

## How to Contribute

### Adding a New Paper
To add a new paper to the repository, please follow these steps:

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Add the paper to the appropriate section in `README.md`.
    *   Follow the existing format: `[Paper Title](URL) | Venue | Year | Notes`
    *   Ensure strict chronological order within sections.
4.  **Commit** your changes with a descriptive message (e.g., `Add Paper Title to Optimization-based methods`).
5.  **Push** to your fork.
6.  Submit a **Pull Request (PR)** to the `main` branch.

### Reporting Issues
If you find any errors or have suggestions for improvements, please open an issue in the repository.

### Style Guide
*   Use standard Markdown.
*   Keep descriptions concise.
*   Ensure all links are working.
